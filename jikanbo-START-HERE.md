# 時間簿 — Claude Code スタートガイド（Web版）

このフォルダの中身を Claude Code に渡せば、そのまま開発に入れます。
**Xcode は不要**です（Web版で進めるため）。Git は Claude Code に任せて構いません（後述）。

---

## 1. このフォルダに置くファイル
すべて同じフォルダ（例：`~/jikanbo`）に置いてください。

| ファイル | 役割 |
|---|---|
| `jikanbo.html` | **ロジックの正本**。動く実装の母体（収支計算・ICS・保存など） |
| `jikanbo-clipboard-mock.html` | **UIの目標形**。クリップボード型の見た目・操作感の見本 |
| `jikanbo-requirements-v1.2.md` | **機能仕様の正本**。何を作るか |
| `CLAUDE.md` | Claude Code が自動で読むルール（実名禁止・単一ファイル維持など） |
| `jikanbo-START-HERE.md` | このファイル。最初の一手 |
| `jikanbo-claude-code-guide.md` | 環境準備・Git超入門（参照用） |

---

## 2. 作業のゴール（方針）
ゼロから作り直すのではなく、**`jikanbo.html` の動くエンジンを土台に、UIを `jikanbo-clipboard-mock.html` の方向（クリップボード型）へ寄せながら、要件 v1.2 の機能を足していく**。

---

## 3. はじめ方（3ステップ）
1. Claude Code を起動（デスクトップアプリでフォルダを開く、またはターミナルで `cd ~/jikanbo && claude`）。
2. 下の「初回プロンプト」をコピーして貼り付ける。
3. Claude Code がまず質問（認識合わせ）を返すので、答えてから実装に進む。

> **Git は覚えなくてOK。** 「ここまでをコミットして」「さっき動いていた状態に戻して」と日本語で頼めば代行されます。詳細は `jikanbo-claude-code-guide.md`。

---

## 4. 推奨ビルド順（Web版）
各ステップは「動く状態」で終え、その都度コミット。順序は目安なので Claude Code と相談して調整可。

- **Step 0：認識合わせ**　要件 v1.2 と現行コードを突き合わせ、解釈のズレを質問として洗い出す（まだ実装しない）。
- **Step 1：データモデルの統合**　要件 §6 の統一属性（contextId / category / tags / importance / urgency / estimate / actual / visibility）へ移行。既存データの移行処理を `load()` に追加し `STORE_KEY` を上げる。
- **Step 2：クリップボードUIへ移行**　固定盤面＋文脈ページャ。文脈ページを可変（追加・削除・並べ替え、〜10枚、名前/種別/色、全予定＝合算）に。**カテゴリ・タグは文字でなくアイコン表示（規定セットから選択・意味づけは可変）。**
- **Step 3：共有用ICS**　公開レベル3段階＋2モード（自分用／共有用）。共有用は非公開を出力しない。
- **Step 4：分析**　カテゴリ/象限/タグ/文脈/週次。無駄の検出（Q3・Q4＋削減候補）と、別軸の「業務改善サイン（Q3過多→仕組み化・委任・発生源対策の提示）」。**閾値は可変＆期間別**。
- **Step 5：見積もり精度**　実績のワンタップ記録 → 見積÷実績（タグ別）→ 補正係数。
- **Step 6：今日ビュー**　今日の経常タスク・余剰時間・朝サマリー → Outlook下書き（mailto）。

※ Step 1〜2 を先に固めると、以降（分析・共有・見積）が同じ器の上に乗るので楽になります。

## 5. 初回プロンプト（これをコピーして貼り付け）

```
これは「時間簿」という、お金の家計簿を時間に応用した個人用Webアプリ（単一HTML）です。
同じフォルダの CLAUDE.md と jikanbo-requirements-v1.2.md を必ず読んでください。
また jikanbo.html（動くロジックの母体）と jikanbo-clipboard-mock.html（UIの目標形）の
両方に目を通してください。

【全体方針】
ゼロから作り直すのではなく、jikanbo.html の動くエンジンを土台に、
UIを jikanbo-clipboard-mock.html の方向（クリップボード型：盤面固定＋文脈をめくる）へ寄せながら、
要件 v1.2 の機能を足していきます。

【まず最初にやってほしいこと（実装はまだしない）】
1. jikanbo.html / jikanbo-clipboard-mock.html / 要件 v1.2 を突き合わせ、
   解釈がズレていそうな点・決め切れていない点を 3〜5 個、質問として挙げてください。
2. そのうえで、jikanbo-START-HERE.md の「推奨ビルド順」をベースに、
   最初の数ステップの具体的な実装計画を提案してください。
3. 私の回答と承認を得てから、Step 1（データモデルの統合）に着手してください。

【守ってほしいこと】
- 単一HTMLファイルのまま（外部依存・ビルド工程なし）。
- データは localStorage。スキーマ変更時は load() に移行処理を入れ STORE_KEY を上げる。
- ドラッグ＆ドロップは Pointer Events 方式（タッチ対応）を維持。
- 配色は :root のトークンを使い、落ち着いたトーンを維持。
- ICS出力ロジックは要件が固まるまで原則さわらない。
- カテゴリ・タグは文字でなくアイコンで表示（規定セット10〜15個から選択、自由入力不可、意味づけはユーザーに委ねる）。
- 全データを一度に描画・計算しない（表示は見えている月・週・文脈の範囲に限定）。
- 顧客名・実名はコードやサンプルに直書きしない（「顧客A大学」等のダミーを使う）。
- 変更の前後で git コミットし、いつでも戻せる状態を保ってください。
```

## 6. 困ったとき
- 動作確認は、編集後の `jikanbo.html` をブラウザで開くだけ（iPadなら「ホーム画面に追加」で全画面のアプリ風に使えます）。
- 壊れたら Claude Code に「直前のコミットに戻して」。
- 方針に迷ったら、要件定義書 v1.2 の §11（ロードマップ）と §6（データモデル）が判断の軸です。
