## 4. 作業の再開方法

Claude Code を起動し、下のプロンプトを貼り付けてください。

```
これは「時間簿」という、お金の家計簿を時間に応用した個人用Webアプリ（単一HTML）です。
CLAUDE.md と jikanbo-requirements-v1.2.md を必ず読んでから作業を開始してください。
現状は CLAUDE.md の「現在の実装状態」と要件定義書 §7 に記載されています。

【今回の目標】
（← ここに今日やりたいことを書く）

【守ってほしいこと】
- 単一HTMLファイルのまま（外部依存・ビルド工程なし）。
- データは localStorage。スキーマ変更時は load() に移行処理を入れ STORE_KEY を上げる（現在 v10）。
- ドラッグ＆ドロップは Pointer Events 方式（タッチ対応）を維持。
- 配色は :root のトークンを使い、落ち着いたトーンを維持。
- ICS出力ロジックは要件が固まるまで原則さわらない。
- qrow.top のレイアウトは display:grid を維持（flex に戻さない。タイトル消えバグの原因）。
- 全データを一度に描画・計算しない（表示は見えている月・週・文脈の範囲に限定）。
- 顧客名・実名はコードやサンプルに直書きしない（「顧客A大学」等のダミーを使う）。
- 変更の前後で git コミットし、いつでも戻せる状態を保ってください。
```

> **Git は覚えなくてOK。** 「ここまでをコミットして」「さっき動いていた状態に戻して」と日本語で頼めば代行されます。

## 5. 困ったとき
- 動作確認は、編集後の `jikanbo.html` をブラウザで開くだけ（iPadなら「ホーム画面に追加」で全画面のアプリ風に使えます）。
- 壊れたら Claude Code に「直前のコミットに戻して」。
- 方針に迷ったら、要件定義書 §11（ロードマップ）と §6（データモデル）が判断の軸です。
