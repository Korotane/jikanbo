# 時間簿 — Claude Code 作成基準・運用ガイド v1.0
作成日：2026-06-25

> このフォルダには3つのファイルを置いてください。
> 1. `jikanbo.html` … 現行アプリ本体
> 2. `jikanbo-requirements.md` … 要件定義書（やりたいことの全体像）
> 3. `CLAUDE.md` … Claude Code が毎回自動で読むルール（これを置くと指示が安定します）

## 0. 結論（ツール選定）

- **Claude Code を使う。** 単一HTMLファイルを反復して育てる作業なので、ファイル編集・ブラウザでの動作確認・バージョン管理ができる Claude Code が最適です。Cowork は非エンジニア向けのナレッジワーク寄りで、コード保守は Claude Code に分があります。
- **作る場所は MacBook Air でOK。** WindowsにはICSだけ渡せば十分です（理由は要件定義書および前回の会話参照：アプリのデータは端末内の `localStorage` に入るため、アプリ自体をWindowsに置く必要はありません）。

## 1. 環境準備（Mac）

### いちばん簡単：デスクトップアプリ（ターミナル不要）
1. https://claude.com/download から **Claude Code デスクトップアプリ（macOS版）** を入手
2. アプリを起動し、Claude.ai アカウントでサインイン
3. 「フォルダを開く」で `jikanbo` フォルダを指定 → そのまま指示を入力

> Gitに不慣れな場合、まずはこのデスクトップアプリが最も摩擦が少ないです。

### ターミナル派：公式インストーラ（推奨・Node不要）
ターミナルを開いて1行：
```bash
curl -fsSL https://claude.ai/install.sh | sh
```
インストール確認：
```bash
claude --version
```
プロジェクトフォルダで起動（初回はブラウザでサインイン）：
```bash
cd ~/jikanbo
claude
```

### 既にNode.jsがある場合（npm・レガシー）
```bash
npm install -g @anthropic-ai/claude-code
```
※ 権限エラー（EACCES）が出たら `sudo` は使わず、npmのグローバル先を自分の所有ディレクトリに変更してください。詳細は公式ドキュメント参照。

### 認証・前提
- **有料プラン（Claude Pro 以上）が必要**です。無料プランでは Claude Code は使えません。
- 不調時は `claude doctor` で診断できます。
- 公式セットアップ手順：https://code.claude.com/docs/en/setup

1. フォルダを用意（例）：

4. 下の「初回プロンプト雛形」を貼って開始

```
これは「時間簿」という、お金の家計簿を時間に応用した個人用Webアプリです。
同じフォルダの CLAUDE.md と jikanbo-requirements.md を読んでから作業してください。
```

## 3. このアプリのコーディング基準

Claude Code には `CLAUDE.md` で同じ内容を渡してありますが、人が理解しておくべき要点を以下に。

### 守ること
- **単一ファイル**：`jikanbo.html` 1枚で完結。外部ライブラリ・CDN・ビルド工程は使わない（オフラインで開けることが価値）。
- **データは localStorage**：スキーマ（保存する項目）を変えたら、`load()` に旧データの移行処理を足し、`STORE_KEY` のバージョン番号を上げる（例 `v7 → v8`）。これを怠ると過去データが消えます。
- **ドラッグ＆ドロップは Pointer Events**：マウス・タッチ統一の現行方式を維持。HTML5の `draggable`/`ondrop` には戻さない（iPadで動かないため）。
- **配色は `:root` のトークンを使用**：落ち着いた低彩度トーンを維持。新しい色も必ずトークン経由で追加。
- **既存機能を壊さない**：特に **ICS出力ロジックは当面さわらない**（最優先で守りたい挙動）。

### ビューの責務（重要：ここが今回の設計の肝）
各画面に役割を1つだけ持たせ、重複させない。

| ビュー | 役割 | 置くもの |
|---|---|---|
| 収支 | 時間の配分 | 固定/定期/投資の費目、月次収支 |
| 年間 | 締切からの逆算 | 本番（ゴール）と準備（マイルストーン＝ガント） |
| カレンダー | 全予定の俯瞰 | すべての予定の表示と日付D&D |
| 4象限 | 単発タスクの仕分け | **単発タスク（tasks）のみ。費目や自動配置カードは置かない** |
| 書き出し | 出力 | ICS、レポート |
| （新規）今日 | 今日の型 | 今日の経常タスク・余剰時間・突発枠（要件 Phase 1） |

### バージョンを刻む
大きな変更の前後で必ずコミット（次章）。前回のGemini版で「壊れて戻せない」状態になったので、ここは徹底します。

## 4. Git 超入門（GitHubアカウントはあるが未経験向け）

### まず結論：Gitは自分で覚えなくてよい
Git＝「セーブ＆ロード」、GitHub＝「そのクラウド保管庫」です。
そして **Claude Code が git の操作を代行できます。** つまり普段は日本語でこう頼むだけで十分です：

- 保存（セーブ）したい：「ここまでの状態をコミットして」
- 戻したい（ロード）：「さっき動いていた状態に戻して」
- 履歴を見たい：「これまでのコミット履歴を見せて」

これだけで「壊れても戻せる」という目的は達成できます。以下のコマンドは“理解のため”と“いざという時の手動操作”用です。

### 最低限の概念と手動コマンド
プロジェクトフォルダで一度だけ初期化：
```bash
git init
```
保存（セーブ。`-m`の後は自分用のメモ）：
```bash
git add -A
git commit -m "ICS出力の動作確認まで完了"
```
状態・履歴の確認：
```bash
git status              # 今どこが変わっているか
git log --oneline       # これまでのセーブ一覧（各行の先頭がID）
```
戻す（ロード）：
```bash
git restore .                          # まだコミットしてない変更を全部捨てる
git checkout <ID> -- jikanbo.html      # 特定のセーブ時点のファイルに戻す
```
※ `<ID>` は `git log --oneline` の先頭に出る7桁程度の英数字。

### GitHub に繋ぐ（バックアップ／別PC共有が必要になったら）
最初はローカルにコミットを刻むだけでも「戻せる」目的は満たせます。クラウド保管が必要になったら：

1. GitHub でリポジトリ（保管庫）を新規作成
2. つなぐ：
   ```bash
   git remote add origin https://github.com/<あなたのID>/jikanbo.git
   git push -u origin main
   ```
   （`main` が無いと言われたら `git branch -M main` を先に実行）

**GitHubの認証**は今はパスワードではなく専用の方式です。未経験なら **GitHub CLI が最も簡単**：
```bash
# GitHub CLI を入れて
brew install gh
# ブラウザでログインして認証を通す
gh auth login
```
この接続作業も「GitHubに繋いでpushして」とClaude Codeに頼めます。

### 重要な注意（データの話）
Gitが保存するのは **コード（jikanbo.html）だけ** です。アプリに入力した時間データはブラウザの `localStorage` にあり、Gitでは保存されません。データのバックアップは別問題なので、将来「設定とデータをJSONで書き出す」機能を足すのが安全です（要件の残課題に記載）。
